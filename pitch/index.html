Neuro-symbolic verification

Every tool call passes through 6 security rules at 4 million operations per second. Before the command ever executes.

🛡️
Dangerous Commands
Detects rm -rf /, dd, fork bombs, shutdown, mkfs before execution
🔀
Path Traversal
Blocks ../ and ..\ sequences that escape the working directory
🔑
Secret Exposure
Catches sk-, ghp_, AKIA patterns and password fields in tool inputs
💾
Destructive SQL
Flags DROP TABLE, TRUNCATE, DELETE FROM in database tool calls
🔁
Repetitive Calls
Warns on 3+ identical calls in last 5 — the doom loop sentinel
💣
Binary Writes
Alerts on >10% non-printable characters in file write operations

Three lines.
That's the tutorial.

No boilerplate. No config files. No providers to set up. Just import, configure, and run.

agent.ts
import { Agent, BashTool, FileReadTool } from "@avasis-ai/synthcode";
import { ollama } from "@avasis-ai/synthcode/llm";
const agent = new Agent({
model: ollama({ model: "gemma3:4b" }), // local, zero cost
tools: [BashTool, FileReadTool],
verifier: new ToolVerifier(), // 6 security rules
});
const { text } = await agent.chat("Fix the failing test in src/utils.ts");
